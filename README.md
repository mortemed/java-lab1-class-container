# Лабораторная работа №1 — Контейнер целых чисел

## Задание

Создать класс-контейнер, позволяющий хранить произвольное количество целых чисел.

Использование встроенных коллекций Java запрещено.

Контейнер должен позволять:

* добавлять элементы;
* получать элементы по индексу;
* удалять элементы.

## Описание реализации

В работе реализован класс `IntContainer`.

Контейнер построен на основе обычного массива `int[]`. Так как размер массива в Java фиксируется при создании, для хранения заранее неизвестного количества элементов используется механизм автоматического расширения.

Когда внутренний массив полностью заполняется, создаётся новый массив большего размера. После этого все существующие элементы копируются в новый массив, а ссылка на внутренний массив заменяется.

Таким образом, пользователь контейнера может добавлять элементы, не указывая заранее их итоговое количество. Фактическое ограничение связано только с доступной оперативной памятью и техническими ограничениями размера массива в Java.

Встроенные коллекции в реализации не используются.

Реализованные операции

Класс IntContainer поддерживает следующие операции:

* add(int value) — добавляет значение в конец контейнера;
* add(int index, int value) — вставляет значение по указанному индексу;
* get(int index) — возвращает значение по индексу без удаления;
* removeAt(int index) — удаляет значение по индексу и возвращает его;
* removeValue(int value) — удаляет первое найденное значение;
* indexOf(int value) — возвращает индекс первого вхождения значения;
* contains(int value) — проверяет наличие значения в контейнере;
* clear() — очищает контейнер;
* size() — возвращает количество элементов;
* isEmpty() — проверяет, пуст ли контейнер;
* toArray() — возвращает копию элементов в виде массива;
* toString() — возвращает строковое представление контейнера.

## Принцип работы динамического массива

В контейнере отдельно хранятся:

внутренний массив elements;
количество реально добавленных элементов size.

Например, внутренний массив может иметь длину 10, но реально в контейнере может храниться только 3 элемента. Остальные ячейки считаются свободными.

При добавлении нового элемента проверяется, есть ли свободное место. Если место есть, элемент записывается в позицию size. Если места нет, вызывается метод расширения массива.

Расширение выполняется следующим образом:

создаётся новый массив в два раза большего размера;
старые элементы копируются в новый массив;
контейнер начинает использовать новый массив;
новый элемент добавляется в контейнер.
